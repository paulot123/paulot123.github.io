ENDEAVR Teledriving System

January 2026 - May 2026

Overview

ENDEAVR Teledriving System is a remote vehicle control prototype built around the CARLA simulator, a React/Electron operator station, and a wheel-based control client. The system streams H.264 video and JSON telemetry over UDP so an operator can monitor speed, gear, braking, signals, and link quality in near real time. It also includes a fleet dashboard for operational visibility and logging, making it a strong exploration of low-latency remote transportation workflows. The project matters because it demonstrates an end-to-end teledriving loop for rural mobility scenarios, from physical input hardware to simulated vehicle response.

Media Gallery

Add `assets/endeavr-teledrive-driving-station.png`
Add `assets/endeavr-teledrive-fleet-dashboard.png`
Add `assets/endeavr-teledrive-architecture.png`

Key Features

  • Built an end-to-end teledriving loop connecting SDL2 wheel input, CARLA simulation, UDP telemetry, and a React/Electron operator station.
  • Streamed low-latency H.264 camera video over UDP with FFmpeg-based encode/decode for real-time remote driving feedback.
  • Implemented a live driving HUD showing speed, gear, brake state, headlights, turn signals, latency, and packet loss.
  • Added a fleet dashboard with operational logs, telemetry snapshots, latency visualization, and system event monitoring.
  • Developed wheel-control networking for steering, throttle, brake, reverse, handbrake, honk, and signal commands over UDP/TCP.
  • Added backend safeguards such as pedal normalization, brake/throttle conflict handling, and a simulated speed cap.
  • Spawned and managed CARLA traffic NPCs plus front and rear camera sensors for richer driving scenarios.
  • Wrote automated Python unit and integration tests covering telemetry formatting, command parsing, and streaming behavior.

Technologies Used

React React Router TypeScript Vite Electron Python CARLA FFmpeg NumPy Pygame Pytest C++ SDL2 UDP TCP

Load Balancer

February 2026 - March 2026

Overview

Load Balancer is a C++ systems simulation that models how incoming network requests are queued, routed to web servers, and processed over discrete clock cycles. The project includes dynamic server scaling based on queue pressure, configurable firewall-style IP blocking, and logging for runtime analysis. It also implements a bonus switch mode that separates processing and streaming traffic across two dedicated load balancers. This makes it a strong demonstration of queue management, simulation design, and core load-balancing concepts.

Media Gallery

Add `assets/LoadBalancerHome.png`
Add `assets/LoadBalancerBonusMode.png`

Key Features

  • Simulated request dispatch across a pool of web servers using discrete clock-cycle execution.
  • Implemented dynamic autoscaling that adds or removes servers based on configurable queue thresholds.
  • Built a configurable firewall layer that rejects requests from blocked IP prefixes to model DOS prevention.
  • Added config-file and command-line controls for server count, runtime, task duration, and request intervals.
  • Logged runtime events and end-of-run statistics including processed, rejected, and queued requests.
  • Developed a bonus switch router that splits processing and streaming jobs across two dedicated load balancers.
  • Used thread-safe queue primitives and modular class design to separate request, server, routing, and config concerns.

VisionEquity

March 2026 - April 2026

Overview

VisionEquity is an early-stage Chrome extension designed to make web content easier to interpret for people with color-vision deficiencies. It applies real-time simulation and correction filters, adds patterns to charts and similarly colored elements, and improves contrast for hard-to-read text. The project matters because it explores practical, page-level accessibility interventions that can help users understand visual information more reliably across the web.
